# AI-Native Architecture

TOP is an AI-native architecture.

This means that the `top/` artifacts — the JSON tree description, implementation prompts,
and related artifacts — are the system's source of truth.
Code is a derivative of these artifacts, generated by AI.

---

## Portability via AI

`top/` can be transferred to another project or another technology
and the implementation can be fully regenerated using AI.

The portable unit is the entire contents of `top/` —
including JSON spec, implementation prompts, and related artifacts.

The implementation code is not transferred and is not intended to be transferred.

Portability is ensured at the level of AI artifacts,
not at the level of platform-specific code.

---

## Implications for architecture

Implementation prompts must be technology-agnostic
where this does not conflict with the task.

Binding to a specific technology in a prompt is permissible,
but must be explicit and deliberate — not accidental.

Behavioral sections of implementation prompts must remain platform-neutral.
Technology-specific details belong in `Platform implementation notes`.
Those notes may help another target technology understand the current implementation's
constraints and edge cases, but they must not be copied mechanically. A new target
must choose its own native mechanism while preserving the platform-neutral behavior.

The spec always remains platform-independent.

---

## Relationship with AI generation

AI-native architecture gives AI sufficient context
to generate correct code without guessing the structure:
- the tree defines the system structure explicitly;
- the prompt defines the node behavior explicitly;
- the human controls the architecture through the spec, not through the code.

This preserves control over the system during intensive AI generation.
