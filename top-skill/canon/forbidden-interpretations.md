# Forbidden Interpretations

The following interpretations are forbidden.

## 1. TOP is a framework

TOP must not be reduced to a framework, runtime library, or UI toolkit.

## 2. Tree is optional

Tree must not be treated as a convenient visualization or as one possible
model among many equally valid alternatives.

In TOP, tree is mandatory.

## 3. TOP is prompt engineering

TOP must not be reduced to prompt templates, agent orchestration, or output
contracts.

These may operationalize TOP, but they are not TOP itself.

## 4. Skill equals paradigm

The skill package must not be confused with the paradigm.

The skill is an execution and guidance layer.
The paradigm is the architectural law.

## 5. AI decides architecture by default

AI must not silently choose architecture, structure, or ownership rules when
those have not been explicitly defined or reconstructed.

## 6. Any tree is a state tree

A tree must not be called a `state tree` unless it contains at least one
`state node`.

## 7. Representation levels can be merged

`spec tree`, `class tree`, and `state tree` must not be collapsed into one
undifferentiated description.

## 8. Working code proves correctness

Compilation or runtime success must not be treated as proof of architectural
correctness.

## 9. Controller and Content are interchangeable

External control boundary and internal content boundary must not be collapsed
into a single undifferentiated node description when TOP modeling requires the
split.

## 10. Local exceptions are harmless

TOP must not be weakened by local exceptions such as:

- “only here”
- “for convenience”
- “the runtime needs it”
- “AI can handle it”

Local structural violations scale into global architectural degradation.
